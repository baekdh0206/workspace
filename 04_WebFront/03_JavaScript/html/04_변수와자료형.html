JavaScript

        function test(arg){
            // 매개 변수 arg : function 지역 변수

                var num5;   // function 지역 변수

변수 선언 방식

        var   : functoin level scope 
                변수(저장하는 값이 변할 수 있음)
                같은 scope에서 변수명 중복 가능(O)
        
        let   : block level scope {}
                변수
                같은 scope에서 변수명 중복 불가능(X)
                (흔히 알고 있는 자바 변수)

        const : block level scope {}
                상수(다시 값 대입 X)
                같은 scope에서 변수명 중복 불가능(X)
                (흔히 알고 있는 자바 상수(final))

        
        1순위 : const
            (JS는 특정 요소를 선택해서 사용하는 경우가 많아
                다른 요소와 구분해서 기억할 수 있도록 
                상수를 이용해서 고정시켜둠)

        2순위 : let

        3순위 : var

JavaScript의 자료형

        자바스크립트는 변수 선언 시 별도의 자료형을 지정하지 않음.
        -> 변수에 대입되는 값(리터럴)에 의해서 자료형이 결정됨!

        - undefined : 정의되지 않음
                    -> 변수를 선언하고 아무런 값도 대입 안했을 때

        - string : 문자열("", '')

        - number : 숫자(정수, 실수)

        - boolean : 논리값(true, false)

        - object : 객체(배열, JS 객체 {K:V, K:V} )

        - function : 함수

        (참고)
        null : 참조하는게 없다(값)

        undefined === null  (=== : 동일한 자료형 + 값인가?)
        -> false (다른거다!!)
